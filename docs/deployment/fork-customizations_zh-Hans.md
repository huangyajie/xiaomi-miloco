# Fork 定制说明

这个 fork 包含一组面向自托管 Docker 部署的修复与界面增强。这份文档已经去除了
所有敏感运行信息，不包含本地 IP、Token、密码或摄像头凭据。

## 已包含的变动

### Home Assistant

- 增加了 Home Assistant 设备列表接口与分组能力。
- 修复了 Home Assistant 设备控制时 `service_data` 透传的问题。
- 增加了 Home Assistant 设备的本地隐藏管理能力：
  - 仅在 Miloco 内隐藏设备
  - 查看已隐藏设备
  - 恢复已隐藏设备
- 修复了设备管理页面里的 Home Assistant 区域信息获取与显示。

### RTSP 摄像头

- 增加了 RTSP 摄像头的 CRUD 接口：
  - 查看已配置 RTSP 摄像头
  - 新增 RTSP 摄像头
  - 编辑 RTSP 摄像头
  - 删除 RTSP 摄像头
- 增加了 RTSP 配置持久化辅助逻辑，避免运行时修改丢失。
- 在 AI 中心左侧摄像头列表中增加了 RTSP 管理入口。
- 优化了 RTSP 编辑行为：
  - 编辑时自动带出已有地址
  - 地址留空时沿用当前流地址

### 设备管理界面

- 增加了米家与 Home Assistant 设备的批量移除能力。
- 增加了“已隐藏设备”恢复入口，支持：
  - 米家与 Home Assistant 分 tab
  - 单个恢复
  - 批量恢复
- 增加了按名称、ID、型号、区域搜索过滤。
- 增加了按区域/家庭分组显示，以及区域级批量选中。

### 摄像头界面

- 为第三方 RTSP 摄像头增加了卡片内联编辑/删除操作。
- RTSP 摄像头卡片优先显示摄像头名称，不再突出通用分组名。
- 优化了全屏视频显示，改为 16:9 横向等比显示，使用黑边留白而不是强行裁切。

## 部署说明

### 推荐部署方式

建议使用 Docker Compose，并为以下内容挂载持久化目录：

- 应用数据目录
- 日志目录
- 配置文件

如果你启用了 RTSP 摄像头运行时管理，请确保 `server_config.yaml` 以可写方式
挂载，否则页面内新增/编辑无法持久化。

### 基本步骤

1. 克隆仓库。
2. 准备 `config/server_config.yaml`。
3. 使用 Docker Compose 启动后端服务。
4. 在 Web 界面完成首次登录初始化。
5. 配置 Home Assistant 和需要使用的云模型。
6. 如有需要，在界面中添加 RTSP 摄像头。

### 安全说明

- 不要把生产环境 Token、API Key、摄像头账号密码或私有网络信息提交进仓库。
- 环境特定的部署参数应放在仓库外部。
- 优先使用环境变量或本地私有配置文件保存敏感信息。

## 运维建议

这份仓库改动刻意不携带本地基础设施细节。如果你需要站点级部署包，
建议把它保存在私有运维仓库或私有部署目录中，而不是公开源码仓库里。

这份说明同样不会包含只用于本地调试、未保留为稳定源码变更的临时规则引擎实验。
